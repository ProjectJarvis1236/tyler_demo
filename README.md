Tyler — это AI-помощник, который помогает управлять персональным компьютером и автоматизировать повседневные задачи. 
В этом репозитории представлена backend-часть проекта. Для публикации были удалены конфиденциальные данные.

Tyler — персональный AI-ассистент, объединяющий возможности языковых моделей, автоматизации и управления компьютером. Он способен не только отвечать на вопросы, но и выполнять реальные действия: искать информацию в интернете, работать с документами Word и Excel, управлять приложениями и окнами, анализировать файлы, использовать систему памяти и взаимодействовать с пользователем в голосовом режиме. Благодаря модульной архитектуре Tyler легко расширяется новыми инструментами и сценариями автоматизации.


## Реализованный функционал

В рамках разработки backend-части Tyler мной были реализованы следующие компоненты:
1. Маршрутизация запросов и взаимодействие с LLM — обработка пользовательских запросов, выбор сценария выполнения, взаимодействие с языковыми моделями и координация вызова внутренних сервисов (`bot_api`, `chat_services`, `llm_selector`).
2. Action System — подсистема выполнения действий, определяющая необходимые инструменты для решения пользовательской задачи (`action_brain`).
3. Управление операционной системой — запуск приложений, управление расположением окон, автоматизация взаимодействия с пользовательским окружением и другое (`app_open`, `windows_manager`, `web_search`, `search_open`, `win_settings`).
4. Работа с офисными документами — создание, анализ и редактирование документов Microsoft Word, электронных таблиц Microsoft Excel и прочих текстовых документов, создание презентаций(`word_manager`, `excel_manager`, `file_manager`, `presentation_manager`).
5. Индексация файлов и система памяти — сканирование файловой системы, индексация пользовательских файлов, реализация различных типов памяти и интеграция семантического поиска на основе эмбеддингов (`file_manager`, `action_memory`, `dialog_memory`, `shortMemory`,).
6. Веб-поиск — получение актуальной информации из сети по запросу LLM (`llm_searching`).
7. Голосовое взаимодействие — распознавание речи пользователя, обработка голосового ввода и поддержка активации по wake word, озвучивание ответа ассистента (`voice_mode`, `tts_api`).
8. Вспомогательные сервисы — создание объектов, парсинг данных и обеспечение взаимодействия между компонентами системы (`creators`, `parsers`).


## Используемые технологии

**Язык программирования:** Python
**Backend:** FastAPI, Uvicorn, WebSocket
**AI и Machine Learning:** Sentence Transformers, ONNX Runtime, Cerebras Cloud SDK
**Векторный поиск:** Qdrant
**Обработка документов:** OpenPyXL, python-docx, pdfplumber, Pandas
**Работа с сетью:** HTTPX, Requests, Beautiful Soup
**Голосовые технологии:** Vosk, Edge-TTS, WebRTC VAD
**Интеграция с ОС:** PyAutoGUI, PSUtil, Watchdog
